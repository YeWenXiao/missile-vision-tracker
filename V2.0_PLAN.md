# 弹载视觉追踪系统 v2.0 实施方案

## 场景定义

- **目标**: 静止不动的地面目标
- **载体**: 飞行中的无人机/弹体，持续接近目标
- **飞行时间**: < 1分钟
- **输入**: 其他飞机提供的**多焦距目标照片集** + 目标标识
- **核心任务**: 从照片集提取目标特征 → 在自己的摄像头画面中找到该目标 → 锁定并持续追踪直到接近

## 硬件平台

| 硬件 | 型号 | 说明 |
|------|------|------|
| 机载计算机 | **NVIDIA Jetson Orin Nano** | 40 TOPS GPU算力, 支持TensorRT加速 |
| 云台相机 | SIYI A8mini | UDP控制, RTSP视频流 1280x720 |
| 视频流 | RTSP H.264 | rtsp://192.168.144.25:8554/main.264 |

## 一、多尺度特征模板（启动时从照片集预提取）

### 照片集输入格式

```
target_photos/
├── far_01.jpg       # 远距离拍摄（目标很小）
├── far_02.jpg
├── mid_01.jpg       # 中距离拍摄（目标轮廓可见）
├── mid_02.jpg
├── near_01.jpg      # 近距离拍摄（目标细节清晰）
├── near_02.jpg
└── target_info.json # 目标标识信息（目标在照片中的标注框等）
```

### 预提取特征（启动时一次性完成，<1秒）

```python
class TargetFeatureBank:
    """从照片集预提取多尺度目标特征"""

    def __init__(self, photo_dir):
        self.templates = []  # 多尺度模板列表

    def load_from_photos(self, photo_dir):
        """启动时加载照片集，提取特征"""
        for photo in sorted(glob(photo_dir + '/*.jpg')):
            crop = load_and_crop(photo)  # 读取照片中的目标区域
            template = {
                'image': crop,                           # 原始裁剪图
                'histogram': calc_hsv_histogram(crop),   # HSV颜色直方图
                'hu_moments': calc_hu_moments(crop),     # Hu矩（形状）
                'size_level': classify_size(crop),       # far/mid/near
                'multi_scale': generate_resized(crop, [0.25, 0.5, 1.0, 1.5, 2.0]),  # 多尺度缩放版本
            }
            self.templates.append(template)
```

### 特征提取方法（4种，轻量级）

| 方法 | 用途 | 耗时 | 特点 |
|------|------|------|------|
| HSV颜色直方图 | 颜色匹配 | ~1ms | 尺度不变，光照敏感 |
| Hu矩 | 形状匹配 | ~2ms | 尺度+旋转不变 |
| 多尺度模板匹配 | 精确定位 | ~5-10ms | 位置精确，但计算量随尺度数增加 |
| YOLO检测 | 快速筛选 | ~20ms | 类别级匹配，配合上述方法确认身份 |

## 二、三阶段状态机

不再需要五阶段（去掉了拍照相关阶段），简化为：

```
SEARCH → TRACK → TERMINAL
 搜索     追踪     末端锁定
```

### 状态转换

```
SEARCH ──找到目标──→ TRACK ──目标占画面>50%──→ TERMINAL
  ↑                    │
  └──丢失超时(5秒)──────┘
```

| 状态 | 触发条件 | 行为 |
|------|----------|------|
| SEARCH | 启动时 / 丢失超时 | 云台扫描 + 每帧用特征模板匹配画面 |
| TRACK | 特征匹配成功 | PID居中 + 自适应zoom + 持续特征验证 |
| TERMINAL | box_ratio > 50% | 最高PID增益，zoom回1x，精确锁定 |

## 三、搜索阶段 — 快速找到目标

### 核心挑战
1分钟内必须快速找到目标，不能花太久搜索。

### 搜索策略

```python
class TargetSearcher:
    """在摄像头画面中搜索目标"""

    def search_frame(self, frame, feature_bank):
        """每帧搜索流程"""

        # 第1步: YOLO检测所有候选目标 (~20ms)
        detections = yolo_detect(frame)

        # 第2步: 对每个检测框，与照片集特征比对
        for det in detections:
            crop = frame[det.y1:det.y2, det.x1:det.x2]

            # 颜色直方图匹配（选最接近的尺度模板）
            color_score = best_histogram_match(crop, feature_bank)

            # 形状匹配
            shape_score = best_hu_match(crop, feature_bank)

            # 综合评分
            score = color_score * 0.5 + shape_score * 0.3 + det.conf * 0.2

            if score > threshold:
                return det  # 找到目标!

        # 没有YOLO检测结果时，用纯模板匹配扫描全画面
        if not detections:
            result = multi_scale_template_match(frame, feature_bank)
            if result.score > threshold:
                return result

        return None  # 未找到
```

### 云台扫描模式
- 起始zoom设为 **最大倍率(6x)** — 远处目标小，需要zoom放大才看得到
- 按预设路径扫描（蛇形/螺旋）
- 如果10秒内6x没找到 → 降到3x扩大视野继续扫描
- 如果20秒内3x没找到 → 降到1x最大视野扫描

## 四、追踪阶段 — 锁定 + 自适应zoom

### PID追踪（复用v1.5）

目标找到后，PID控制云台保持目标居中。参数随距离自适应：

| 目标大小 (box_ratio) | Kp | Ki | Kd | 说明 |
|---------------------|----|----|-----|------|
| < 5%（远） | 12 | 0.1 | 1.0 | 低增益，目标移动慢 |
| 5%-20%（中） | 20 | 0.2 | 2.0 | 中增益 |
| 20%-50%（近） | 30 | 0.3 | 3.0 | 高增益，快速响应 |
| > 50%（末端） | 40 | 0.3 | 4.0 | 最高精度 |

### Zoom策略（接近过程自动调整）

目标是静止的，载体在接近，目标会越来越大：

```python
def adaptive_zoom(box_ratio, current_zoom):
    """根据目标大小自动调整zoom"""
    if box_ratio < 0.03:
        target_zoom = 6    # 远处：最大zoom看清目标
    elif box_ratio < 0.08:
        target_zoom = 4    # 中远：适度zoom
    elif box_ratio < 0.15:
        target_zoom = 3    # 中距离
    elif box_ratio < 0.30:
        target_zoom = 2    # 中近
    else:
        target_zoom = 1    # 近距离/末端：不需要zoom

    # 平滑过渡，不要跳变
    if target_zoom < current_zoom:
        do_zoom_out_step()
```

**关键：zoom只需要单向递减（6→4→3→2→1），因为载体一直在接近。**

### 持续特征验证（防跟错）

追踪过程中每秒做一次特征验证：

```python
def verify_target(current_crop, feature_bank):
    """验证当前追踪的是否还是原目标"""
    color_score = best_histogram_match(current_crop, feature_bank)
    if color_score < 0.3:
        # 可能跟错了！降低置信度，准备切换回搜索
        return False
    return True
```

## 五、丢失恢复策略

目标静止 + 有照片集特征 → 恢复比v1.5简单得多：

1. **立即zoom缩小到1x** — 扩大视野
2. **回到最后锁定的云台角度** — 目标没动，还在那个方向
3. **用特征模板重新搜索** — 颜色+形状匹配，不纯依赖YOLO
4. **3秒未恢复 → 小范围螺旋搜索**（目标附近±10°）
5. **10秒未恢复 → 全局重新搜索**

## 十一、关键风险与应对

| 风险 | 应对 |
|------|------|
| 照片集拍摄角度与本机视角差异大 | 颜色直方图对视角变化鲁棒；多张不同角度照片增加覆盖 |
| 场景中有多个相似目标 | 多特征融合提高区分度；照片集标注精确的目标特征 |
| 远距离目标太小检测不到 | 搜索阶段先zoom到6x放大视野中心区域 |
| zoom切换时短暂丢失 | 记录锚定角度，zoom变化后PID继续控制 |
| YOLO模型不认识新目标 | YOLO只做"这里有东西"的粗筛，身份确认靠特征匹配 |
